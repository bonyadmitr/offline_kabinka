# Спецификация: офлайн-PWA «Туалеты Минска» (offline_kabinka)

- **Дата:** 2026-06-26
- **Статус:** утверждено к реализации
- **Рабочая папка:** `/Users/y.bondar/Downloads/OfflineMaps/app_web_offline_kabinka` (свой git-репозиторий)
- **Деплой:** `https://bonyadmitr.github.io/offline_kabinka` (GitHub Pages, base-path `/offline_kabinka/`)
- **Источник данных/фото:** API `https://kabinka.by` (см. `../../../docs/API.md`, `../../../docs/kabinka-map-ui.md`)

## 1. Цель и охват

Статический PWA-просмотрщик общественных туалетов Минска, работающий **офлайн** после
первого скачивания: карта (векторные OSM-тайлы), 263 точки, превью фото, фильтры,
детальные карточки. Основной таргет — **iOS-телефон**, десктоп — «нормально».

**Только просмотр.** Никакой отправки данных в API (отзывы/добавление точек —
out of scope, см. §17).

## 2. Принятые решения

| Тема | Решение | Причина |
|---|---|---|
| Движок карты | **MapLibre GL JS + PMTiles** (вектор) | меньше вес, чётко на retina, рантайм-переключение языка карты RU↔EN без перекачки, тёмная тема |
| Маршруты | **Гибрид**: офлайн — линия+расстояние+компас; онлайн — deep-link в Яндекс/Google/Apple | офлайн turn-by-turn даёт маргинальную пользу для «ближайший туалет» при большой цене (вес+сложность) |
| Объём | **read-only просмотрщик** | проще, безопаснее, без модерации |
| Деплой | `gh` (авторизован локально), GitHub Pages | по требованию |
| Формат данных | вшиваем **`locations.json` как есть** (полные поля деталей + пути к фото + инлайн-комментарии) | комментариев всего 7, отдельный файл — лишнее усложнение |
| Хранение карты | `.pmtiles` **не в git**; разово в `/Users/y.bondar/Downloads/OfflineMaps/minsk_map`, в `dist/` попадает на деплое | репозиторий исходников лёгкий |
| Стек | Vite + TypeScript, **без UI-фреймворка** (vanilla + модули + мини-стор) | лёгкий бандл, дружелюбно к памяти iOS |
| Тесты | Vitest (юнит) + Playwright (UI) | полезные тесты, не цифра ради цифры |

## 4. Данные

- **Вшитый baseline:** `public/data/locations.json` (копия `OfflineMaps/data/locations.json`,
  263 точки, ~672 КБ, gzip ~120 КБ). Содержит всё: поля деталей, `photos[].{remote,url,thumb}`,
  инлайн-`comments`.
- **Рантайм:** при старте приложение использует обновлённые данные из **IndexedDB**, если
  они есть и новее baseline; иначе baseline.
- **Считаем на клиенте (офлайн-безопасно):**
  - **«Открыто сейчас»** — по **Europe/Minsk** (UTC+3, без DST с 2011). Текущий момент
    берём с устройства (`Date`), переводим в минскую TZ через
    `Intl.DateTimeFormat('ru-RU',{timeZone:'Europe/Minsk',weekday,hour,minute,hour12:false})`,
    сравниваем с `working_hours` (они в минском локальном времени). Корректно при любой
    таймзоне телефона. Учитываем перерывы (`break_start/end`), `is_closed`, переход через
    полночь (напр. 22:00–06:00) и круглосуточный режим.
  - **Расстояние/сортировка** — Haversine от геопозиции (`distance_meters` сервера игнорим).

### 4.1 Единый скрипт обновления данных (`build-data.mjs`)

Объединяет `fetch_data.sh` + `set_photo_paths.sh` + `export_variants.sh`. Тянет из API:
список (`per_page=500`) → детали по точкам → комментарии. Проставляет пути к фото. Пишет
`public/data/locations.json`. **Сбой комментариев не фатален** (try/catch, остальное
сохраняется). Используется при сборке/деплое для свежего baseline.

### 4.2 In-app «Обновить данные» (настройки)

Тот же конвейер в браузере (CORS открыт). Шаги:
1. `GET /locations?per_page=500` → апсерт summary-полей всех точек.
2. По точкам: `GET /locations/{id}` (детали) и `GET /locations/{id}/comments` —
   прогрессивно, с вежливой задержкой. **Комментарии в отдельном try/catch** — упали,
   оставляем локации.
3. Мерж комментариев инлайн в объекты локаций (форма как у baseline), запись в IndexedDB.

**Прогресс (п.5):** видимый бар «Обновлено N из M точек», этапы (список → детали →
комментарии). Прерывание/закрытие — сохраняем что успели.

**Diff-отчёт по итогу (п.6):** сравнение старого и нового набора по `id`. Сообщаем:
«Добавилось X, убралось Y» (+ опц. «обновлено Z»), либо **«Новых данных нет»**.

## 5. Карта (PMTiles)

- **Сборка `minsk.pmtiles`** (`build-map.sh`, локально/разово, выход в `minsk_map/`):
  экстракт Беларуси (Geofabrik) → обрезка боксом Минска (lat 53.80–54.00, lon 27.35–27.75)
  → **Planetiler** (схема OpenMapTiles: есть `name:ru`/`name:en`/`name:be`) → `minsk.pmtiles`
  + `map-version.json` (версия/дата/размер/хэш).
- **Подписи RU** (п.5): в стиле `text-field: ["coalesce",["get","name:ru"],["get","name"]]`.
  **Переключение на EN** (п.12) — смена выражения языка в рантайме, без перекачки.
- **Тёмная тема:** отдельный набор paint-свойств.
- **Оптимизация веса** (п.13): обрезка Минском; `maxzoom=16`; выкинуть лишние слои; упрощение.
  Оценка **15–40 МБ**, уточняется реальной сборкой; пост-оптимизация — §15.
- **Память iOS:** `maxBounds`=Минск, ограниченный `maxzoom`, очистка GL при уходе в фон.
  `.pmtiles` качается в **OPFS**, читается диапазонами через `createSyncAccessHandle()` в
  Web Worker (так требует WebKit, см. `../../../docs/offline-maps-storage.md`). Раздаётся
  приложению same-origin из `dist/map/`.

### 5.1 In-app «Обновить карту» (настройки, п.4)

Карта на сайте лежит same-origin: `…/offline_kabinka/map/minsk.pmtiles` + `map-version.json`.
Кнопка: `GET map-version.json` → сравнить с версией в OPFS. Совпало → **«Обновлять
нечего»** сразу. Иначе скачать новый `.pmtiles` (потоково, **с прогрессом**), заменить в
OPFS, обновить версию. Ошибки — с кодами (§9). Свежую карту на сайт отправляет тот же
`build-map.sh` (выход) + `deploy.sh`.

## 6. Офлайн-стратегия и хранилище

- **Service Worker** прекэширует оболочку (HTML/JS/CSS/иконки) → мгновенный офлайн-старт.
- **Тяжёлые ассеты** (`minsk.pmtiles`, пакет превью, данные) качаются отдельно **с
  прогресс-баром** в OPFS/Cache — на первом запуске и повторно из настроек.
- **Пакет превью:** 1088 файлов пакуем в **`thumbs.bin` + `thumbs-index.json`**
  (`pack-thumbs.mjs`) — один потоковый запрос с прогрессом вместо 1088. Раздаём blob-URL.
- **iOS-нюанс:** долго живёт только установленный на «Домой» PWA (иначе Safari чистит
  через 7 дней — см. `../../../docs/ios-pwa-limitations.md`). Показываем **закрываемую
  подсказку** «Поделиться → На экран „Домой“» и дублируем в настройках (п.2); запрашиваем
  `navigator.storage.persist()`.
- **Размер приложения в настройках (п.10):** `navigator.storage.estimate()` + разбивка по
  бакетам (карта/превью/данные/оболочка/временный кеш).
- **«Очистить кеш (X МБ)» (п.11):** чистит **временное** — полноразмерные фото из
  `/storage/...` и temp апдейта; **не трогает** офлайн-пакет. Отдельно — «Переустановить
  офлайн-пакет». Размер показан на кнопке.

## 7. UI / экраны

- **Карта (главный экран):** полноэкранный MapLibre; слева сверху ⚙️ и «Фильтры»;
  **зум +/− в т.ч. на мобиле** (улучшение); геолокация (точка+точность, рецентр); маркеры
  по цвету `price_type` с числом рейтинга; кластеризация MapLibre.
- **Список/деталь:** мобайл — 3-state bottom-sheet; десктоп — боковая панель.
- **Карточка** (п.15,19): галерея превью (свайп; тап → полноэкран) → название+бейдж+адрес →
  рейтинг 3 оси + общий → чипы (♿/кабины/писсуары/раковины) → теги → режим работы
  (сегодня подсвечено, «открыто сейчас» локально) → цена → **кнопка «Маршрут»** (§8) →
  комментарии (чтение) → «Сообщить об ошибке» (онлайн) → шаринг-ссылка.
- **Фильтры** (п.22): открыто сейчас · пол (`layout_type`) · оплата · доступность · удобства
  (11 тегов) · мин. рейтинг. **Применяются на клиенте, мгновенно, офлайн** (улучшение).
- **Настройки** (п.6): язык UI RU/EN и язык карты RU/EN (независимо, п.12) · тёмная тема ·
  «Обновить данные» (прогресс/итог) · «Обновить карту» · радиус списка · навигатор по
  умолчанию · размер приложения · «Очистить кеш» · подсказка установки · device-id · версия.
- **Поиск** (п.2, улучшение): по названию/адресу, офлайн, локальный индекс.
- **Шаринг** (п.2): `#id=<id>` в URL открывает карточку точки; кнопка «Поделиться»
  (Web Share API / копировать ссылку).
- **Просмотрщик фото:** полноэкран, свайп, пинч-зум; полноразмерное грузится из сети
  обычным `<img>` (без `X-Device-ID`, см. §10), кэш — временный.

## 8. Маршруты — гибрид (п.14)

Интерфейс `Router { route(from, to): RouteResult }`, чтобы позже подключить офлайн-роутер
без переделок. Реализации:
- **LineRouter (офлайн):** геодезическая линия от пользователя к точке на карте +
  расстояние + стрелка-компас (DeviceOrientation, с фолбэком если нет доступа/разрешения).
- **DeeplinkRouter (онлайн):** меню «Маршрут» → Яндекс Карты / Яндекс Навигатор / Google /
  Apple (deep-link, навигатор по умолчанию из настроек).

## 10. Фото (п.11, 20)

Вшитые **превью** (`thumbs/`, 307×173, ~10 МБ) в карточке/галерее. Тап → полноэкран →
**полноразмерное** из `https://kabinka.by/storage/locations/{id}/photo_N.jpg` обычным
`<img>`. **Отклонение от п.20:** `X-Device-ID` к фото не добавляем — для `/storage` не
нужен, а с ним CORS-preflight отклонится. `X-Device-ID` (один UUID, генерится один раз,
хранится в localStorage) используется для API-запросов обновления.

## 11. i18n (п.5, 12)

Два независимых переключателя: язык UI и язык карты. UI-строки в словарях (`ru` полный,
`en` — каркас структуры, дозаполняется). Контент данных (названия/адреса) остаётся на
языке источника (русский) — топонимы не переводим; названия тегов есть на RU, EN позже.
Подписи карты переключаются выражением языка в стиле.

## 12. Деплой (п.18)

`git init`; `vite base='/offline_kabinka/'`. Карта и пакет превью **не в git** (.gitignore).
`deploy.sh`: `vite build` → копировать в `dist/` данные, `thumbs.bin`/index, `minsk.pmtiles`
+ `map-version.json` из `minsk_map/` → опубликовать на Pages (`gh`/Pages-артефакт; ветка
публикации — single-commit/force, исходный `main` чист). Проверить `gh auth status` и что
аккаунт = `bonyadmitr` перед пушем. (Если Release-asset + CORS+range подтвердятся рабочими —
рассмотреть как полностью git-free альтернативу для карты.)

## 13. Бюджет размера (п.21)

Оболочка (сотни КБ) + данные (~0.7 МБ) + превью (~10 МБ) + карта (~15–40 МБ) ≈ **30–50 МБ**.
Для установленного iOS-PWA (квота ~60% диска) — норм; в вебе работает онлайн с дозагрузкой.
Прогресс скачивания показываем (§6).

## 14. Тестирование (п.8)

**Полезные тесты, не ради цифры.** Vitest (юнит) + Playwright (UI/e2e, headless).

- **Юнит:** `open-now` (таймзона Минска, перерывы, переход через полночь, круглосуточно,
  закрыто) · Haversine/сортировка · фильтрация (все комбинации флагов/тегов/рейтинга) ·
  diff обновления (added/removed/changed) · сравнение версий карты · распаковка
  `thumbs.bin` по индексу · парсинг/нормализация данных · routing-математика (азимут,
  расстояние, формат deep-link) · i18n (нет пропущенных ключей в `ru`).
- **UI/e2e:** старт офлайн (моки ассетов) · открытие карточки по клику · применение
  фильтров меняет список/маркеры · открытие/закрытие настроек · состояния прогресса
  обновления данных/карты · показ ошибок с кодами (мок offline/5xx) · просмотрщик фото ·
  шаринг-ссылка `#id` открывает карточку · поиск.

## 19. Допущения и риски

- Точную массу `.pmtiles` подтвердит реальная сборка (диапазон 15–40 МБ).
- Деплой карты не-в-git: основной путь — публикация `dist/` (с картой) на Pages; запасной —
  Release-asset + рантайм-загрузка (зависит от CORS/range у GitHub — проверить).
- Для `build-map.sh` нужен Java 21+ (Planetiler) и ~сотни МБ экстракта Беларуси (разово,
  кэшируется). При отсутствии Java — поставить через brew или взять tilemaker.
- MapLibre GL на iOS Safari чувствителен к памяти — держим `maxzoom`/`maxBounds`, чистим GL.
